##### 十分な達成方法

##### 失敗例

- [F110: 達成基準 2.4.11 隠されないフォーカス (最低限) の失敗例 ― フォーカスされた要素を完全に隠す固定フッター又はヘッダー](https://waic.jp/translations/WCAG22/Techniques/failures/F110)

#### [2.4.12 隠されないフォーカス (高度)](https://waic.jp/translations/WCAG22/Understanding/focus-not-obscured-enhanced)

適合レベル AAA

##### 十分な達成方法

- [C43: CSS の scroll-padding を用いてコンテンツを隠さないようにする](https://waic.jp/translations/WCAG22/Techniques/css/C43)

##### 失敗例

- キーボードフォーカスを持つコンポーネントの上にコンテンツを表示させ、フォーカスインジケータの一部を視覚的に覆うインタラクション。この動作は、利用者がカタログ内をナビゲートするときに、製品に関する詳細情報を提供することを目的とした広告又は販促資料で遭遇する可能性がある。
- ページにはスティッキーフッター（ビューポートの下部に貼り付けられる）がある。ページをタブで下に移動すると、ビューポート内のコンテンツが十分な scroll padding なしにスクロールするため、フォーカスされた項目がフッターによって部分的に隠される。

#### [2.4.13 フォーカスの外観](https://waic.jp/translations/WCAG22/Understanding/focus-appearance)

適合レベル AAA

- [G195: Using an author-supplied, visible focus indicator](https://w3c.github.io/wcag/techniques/general/G195)
- [C40: Creating a two-color focus indicator to ensure sufficient contrast with all components](https://w3c.github.io/wcag/techniques/css/C40)
- [C41: コンポーネント内に堅固なフォーカスインジケータを作成する](https://waic.jp/translations/WCAG22/Techniques/css/C41)

##### 失敗例

- 改行可能なインラインテキストに CSS ボーダーを使用する
